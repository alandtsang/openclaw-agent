# OpenClaw-Agent Project Guidelines

This file (`CLAUDE.md`) contains the core architectural, coding, and environmental principles specific to this project. As an AI Agent, refer to this file to understand the current engineering context and constraints. 

## Project Setup & Environment
- **Runtime**: Node.js and TypeScript environment.
- **Agent Framework**: Google Agent Development Kit (ADK).
- **Package Manager**: Use `npm` for installing dependencies and running scripts.
- **Entry Structure**: The root of execution typically sits within `src/index.ts` or specific test files (e.g., `test_dynamic.ts`, `test_codes.ts`).

## Code Structure & Philosophy
- `src/agent`: Core logic, instructions, and agent definitions.
- `src/tools`: Built-in utilities and fundamental tools provided to the agent.
- `src/services`: Integrations with external APIs (e.g., Feishu).
- `skills/`: The primary location for dynamic capabilities (installed or auto-generated by the Agent). Each skill here must follow the standard structure (`index.ts` containing the ADK `FunctionTool` export, and `SKILL.md` for LLM prompt context).

## Memory Usage Instructions
- `MEMORY.md`: This file is your primary index to user facts, preferences, and long-term history. 
- **Sub-Memories**: If `MEMORY.md` becomes excessively long or detailed (e.g., tracking a complicated bug context or storing rigid API documentation), use your tools to **create a standalone file** (like `topic-name.md` or `memory/api-guidelines.md`). Then, leave a brief summary and the file index path in `MEMORY.md`. 
- `SOUL.md`: Your core identity and behavioral rules.

## Formatting & Behavior
- Prefer concise, idiomatic TypeScript code.
- Always use `zod/v4` schema validations for parameters inside new/modified `FunctionTool` implementations. 
- Prefer robust error handling over crashing scripts.
