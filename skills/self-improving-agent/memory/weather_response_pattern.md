# 模式：基于工具内部指令的响应优化

**情景**: 当调用某些工具（例如 `weather_tool`）时，工具的返回结果可能包含一个 `note` 或类似字段，其中提供了关于如何组织和呈现响应的额外指导。

**可复用模式**:
1.  **识别指导字段**: 在处理工具返回结果时，主动检查是否存在包含用户交互指导的字段（如 `note`）。
2.  **提取并应用指导**: 根据这些指导，调整 Agent 的回复语气、内容组织和信息呈现方式。这可以包括：
    *   使用特定语言（如中文）。
    *   融入情感（如“带有感情地汇报”）。
    *   突出特定信息（如健康建议）。
3.  **提升用户体验**: 通过遵循这些指导，Agent 能够提供更贴近用户需求、更人性化和更易于理解的回复，从而显著提升用户体验。

**经验教训**: 将用户交互的考虑前置到工具设计中，通过工具返回的辅助信息来指导 Agent 的后续响应生成，是一种高效且可扩展的优化方式。这减少了 Agent 在生成响应时的不确定性，并确保了输出的一致性和高质量。
